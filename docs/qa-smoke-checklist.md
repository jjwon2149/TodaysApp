# DailyFrame QA Smoke Checklist

이 체크리스트는 기능 완료 후 핵심 회귀를 빠르게 확인하기 위한 표준 smoke runbook이다. 전체 탐색 QA가 아니라, 새 변경 사항이 DailyFrame의 기본 기록 루프를 깨지 않았는지 확인하는 데 집중한다.

## 실행 기준

- 대상: iOS 18.5+ simulator 또는 device, 최신 브랜치 빌드
- 데이터: 새 설치 상태에서 1회, 기존 기록 1개 이상 있는 상태에서 1회 실행
- 증거: 실패 항목은 화면, 재현 단계, 기대/실제 결과를 이슈 댓글에 남긴다.
- 제외: 백업, 위젯, 접근성 전용 상세 케이스는 별도 QA 이슈에서 다룬다.

## Core Smoke

1. 앱 실행
   - 새 설치에서 앱이 크래시 없이 실행된다.
   - 온보딩 완료 후 홈에 진입하고, 재실행 시 온보딩이 반복 노출되지 않는다.

2. 새 기록 생성
   - 홈의 primary CTA에서 기록 작성 화면으로 1탭 안에 진입한다.
   - 사진 없이 저장할 수 없고, 메모/감정 없이 사진만으로는 저장된다.
   - 사진 선택 또는 촬영 후 저장하면 완료 화면이 표시된다.

3. 홈 반영
   - 저장 직후 홈이 completed 상태로 바뀐다.
   - 오늘 기록 카드, 최근 기록, 이번 달 진행률이 새 기록을 반영한다.

4. 미션
   - 오늘의 미션 제목/프롬프트가 홈에 표시된다.
   - 오늘 기록 저장 후 미션 완료 상태가 홈과 완료 화면에 반영된다.
   - 같은 날짜에 앱을 재실행해도 미션이 바뀌지 않는다.

5. 캘린더
   - 오늘 날짜 셀에 저장한 사진 썸네일이 표시된다.
   - 월 이동 후 돌아와도 썸네일과 월간 기록률이 유지된다.
   - 날짜 셀을 탭하면 해당 기록 상세로 이동한다.

6. 기록 수정
   - 상세 화면에서 수정 진입이 가능하다.
   - 메모 또는 감정을 수정하면 상세, 홈, 캘린더 진입 경로에서 동일한 값이 보인다.
   - 이미지를 다시 선택한 경우 새 이미지가 보이고 이전 이미지가 노출되지 않는다.

7. 기록 삭제
   - 삭제 전 확인 다이얼로그가 표시된다.
   - 삭제 후 캘린더 썸네일과 홈의 오늘 완료 상태가 제거된다.
   - 삭제한 기록은 앱 재실행 후에도 다시 나타나지 않는다.

8. Streak / Freeze
   - 오늘 첫 기록 저장 후 current streak와 longest streak가 증가 또는 유지된다.
   - 연속 날짜 데이터에서는 streak가 이어지고, 하루 누락 데이터에서는 freeze가 1회 차감되거나 정책대로 reset된다.
   - 기록 삭제 후 streak가 남아 있는 활성 기록 기준으로 재계산된다.

9. 알림 설정
   - 프로필에서 알림 on/off와 알림 시간 변경이 가능하다.
   - 권한 거부 상태에서도 앱 사용, 기록 저장, 캘린더 조회가 막히지 않는다.
   - 권한 허용 상태에서 알림 시간을 바꾸면 예약 상태 문구가 갱신된다.

10. 앱 재실행
    - force quit 후 재실행해도 기록, 미션 완료, 캘린더, streak, 알림 설정이 유지된다.
    - 저장된 이미지 또는 썸네일이 누락되어도 앱이 크래시하지 않고 fallback UI를 보여준다.

## 대표 실패 경로

1. 사진 권한 또는 카메라 권한 거부 후 대체 경로나 설정 안내가 표시되는지 확인한다.
2. 저장 도중 네비게이션을 닫거나 다시 저장을 눌러도 중복 저장 또는 깨진 상태가 생기지 않는지 확인한다.
3. 삭제 후 뒤로 가기, 홈 복귀, 캘린더 재진입을 반복해 stale record가 보이지 않는지 확인한다.
